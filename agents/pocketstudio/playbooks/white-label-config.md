# White-Label Configuration — Pocket Studio 2.0

Version: 1.0
Last updated: 2026-03-27

## Overview

All Pocket Studio agent output is white-label by default. No agent should ever reference "Pocket Studio," its internal agent names, or Paperclip infrastructure in client-facing deliverables. Client brand identity flows into every output through the `brand-dna.yaml` configuration.

## White-Label Rules

### What agents must never include in client-facing output
- "Pocket Studio" or any variation
- Internal agent names (Scribe, Sentinel, Strategist, etc.)
- Paperclip issue IDs, project IDs, or system references
- Internal workflow details (heartbeats, approval gates, agent handoffs)
- References to AI generation ("generated by AI," "AI-powered," etc.) unless the client's brand DNA explicitly permits it

### What agents must always include
- Client's brand name (from `brand_voice.name` in brand-dna.yaml)
- Client's approved disclaimers (from `compliance.mandatory_disclaimers`)
- Proper ASCI disclosures on sponsored/paid content
- Client-specified attribution format if defined

## Brand DNA Integration

Every agent loads brand context from `clients/[client-slug]/brand-dna.yaml` (see `multi-client-architecture.md` for schema). The following fields drive white-label output:

### Copy (Scribe, Diplomat, Publicist)
| brand-dna.yaml field | How it's used |
|---------------------|---------------|
| `brand_voice.tone` | Sets writing tone (professional, playful, bold) |
| `brand_voice.language` | Primary language + code-switching rules |
| `brand_voice.formality` | Register (formal for BFSI, casual for D2C) |
| `brand_voice.personality_traits` | Adjectives that guide word choice |
| `brand_voice.avoid` | Blacklisted phrases and patterns |
| `client.name` | Always used instead of any internal name |

### Visual (Visualist, Builder)
| brand-dna.yaml field | How it's used |
|---------------------|---------------|
| `visual_identity.primary_colors` | Dominant colors in all graphics |
| `visual_identity.secondary_colors` | Accent colors |
| `visual_identity.fonts.heading` | Heading font in all assets |
| `visual_identity.fonts.body` | Body text font |
| `visual_identity.logo_url` | Logo placement on all branded assets |
| `visual_identity.style` | Overall aesthetic (minimalist, bold, etc.) |

### Compliance (Sentinel)
| brand-dna.yaml field | How it's used |
|---------------------|---------------|
| `compliance.asci_sectors` | Triggers sector-specific audit rules |
| `compliance.mandatory_disclaimers` | Required on all public-facing output |
| `compliance.prohibited_claims` | Hard blocks — Sentinel rejects if present |
| `compliance.required_disclosures` | Must appear on specified content types |

### Reporting (Diplomat, Analyst)
| brand-dna.yaml field | How it's used |
|---------------------|---------------|
| `client.name` | Report headers, subject lines |
| `visual_identity.primary_colors` | Report styling and charts |
| `visual_identity.logo_url` | Report cover and headers |

## Agent Prompt Conventions

Agent AGENTS.md files are role-generic and must not contain client-specific references. Client context is injected at runtime:

```
Task assigned → Agent reads projectId → loads brand-dna.yaml → applies to output
```

Each agent's AGENTS.md includes a standard section:

```markdown
## Brand Context Loading

When working on a client task:
1. Read the task's project workspace path
2. Load `brand-dna.yaml` from the workspace
3. Apply brand voice, visual identity, and compliance rules to all output
4. Never reference internal Pocket Studio names or processes in deliverables
```

## Client-Specific Overrides

Some clients may need overrides beyond the base brand-dna.yaml:

### AI Attribution
```yaml
# In brand-dna.yaml
white_label:
  allow_ai_attribution: false          # default: false
  attribution_text: null               # if true: "Powered by [text]"
  agency_attribution: "by [Agency]"    # optional: client's own agency branding
```

### Custom Templates
Clients can provide custom output templates in their workspace:
```
clients/[client-slug]/
├── brand-dna.yaml
├── templates/
│   ├── social-post.md          # custom social post format
│   ├── report-header.md        # custom report header
│   └── email-signature.md      # custom email sign-off
└── assets/
    ├── logo.png
    └── brand-guide.pdf
```

Agents check for custom templates before using defaults. If `templates/social-post.md` exists, Scribe uses it instead of the standard output format.

## Validation Checklist

Before any deliverable goes to a client, Sentinel verifies:

- [ ] No "Pocket Studio" references
- [ ] No internal agent names
- [ ] No Paperclip system references
- [ ] Client brand name used consistently
- [ ] Correct brand colors/fonts (for visual assets)
- [ ] All mandatory disclaimers present
- [ ] ASCI sector rules applied
- [ ] AI attribution matches client's `allow_ai_attribution` setting
